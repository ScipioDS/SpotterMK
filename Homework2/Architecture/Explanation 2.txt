Register Controller - Register Service - User Repository  
	When registering a user, the Register Controller will receive the request from the MVC Controller and forward it to the Register Service. In this service, the validity of the 	information will be checked and the information will be saved to the database. The User Repository will handle saving the data in the database.

Login Controller - Login Service - User Repository 
	When a user logs in, the Login Controller will receive the request from the MVC Controller and forward it to the Login Service. In this service, the user's authentication and 	authorization will be performed, with the User Repository being used to retrieve information from the database. All of this will be enabled through Spring Security.

Search Controller - Search Service - LSTM Service - Company/Historical Data Repository  
	When searching for companies, the Search Controller will receive the request from the MVC Controller and forward it to the Search Service. This service will contain the main business 	logic for the application. The company and its historical data will be selected. The data retrieval will be performed with the help of the Company and Historical Data 	Repository. 	After that, a request will be sent to the LSTM Service, which will make a prediction of the stock price for the next month for the selected company.

Technologies used:  
	Frontend - Bootstrap, Thymeleaf  
	Backend - Spring Boot and Python  
	Database - Postgres
